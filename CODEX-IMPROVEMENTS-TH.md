# รายงานปรับปรุงทันใจ AI Studio

## สิ่งที่ปรับในชุดนี้

- ปิด Local Auth fallback ที่ใช้รหัสผ่านใดก็ได้โดยค่าเริ่มต้น
- ปิดปุ่มสมัครสมาชิกสาธารณะโดยค่าเริ่มต้น บัญชีใหม่ควรสร้างโดยเจ้าของระบบ
- เพิ่มข้อความแจ้งเมื่อเชื่อมต่อระบบสมาชิกไม่ได้ และสถานะกำลังเข้าสู่ระบบ
- เพิ่มปุ่มแสดง/ซ่อนรหัสผ่าน พร้อมกำหนดขั้นต่ำ 6 ตัวอักษร
- เพิ่มปุ่มเปิดเมนูมือถือที่ขาดหายไป พร้อมสถานะเปิด/ปิดและคืนโฟกัส
- เพิ่ม Skip link, focus ring, aria-live และสถานะเมนูปัจจุบัน
- ทำให้การ์ดเมนูที่ไม่ใช่ปุ่มใช้งานด้วย Enter/Space ได้
- แก้ event นำทางที่เคยทำงานซ้ำสองครั้ง
- ป้องกันการคัดลอกและบันทึกผลลัพธ์ว่าง
- เพิ่มคำยืนยันก่อนล้างโปรเจกต์ในเครื่อง
- ซ่อนปุ่มล้างข้อมูลในเครื่องเมื่อกำลังใช้ Cloud project
- ป้องกันหน้าโปรเจกต์พังเมื่อข้อมูล Local Storage เสียหาย
- แก้ช่อง XSS จากชื่อไฟล์ภาพในหน้าพรีวิว และคืนหน่วยความจำของ Object URL
- จำกัดลิงก์ที่เปิดจากระบบให้เป็น HTTP/HTTPS
- เพิ่ม Reduced Motion และปรับ Toast/Topbar บนมือถือ

## สิ่งที่ควรทำต่อ (เรียงลำดับความสำคัญ)

### 1. ทำ Access Control ที่ฝั่งระบบจริง

หน้า Login บน Static Web ป้องกันข้อมูลใน Firestore ได้ก็ต่อเมื่อ Firestore Rules ตรวจ `uid` ถูกต้อง แต่ไม่สามารถซ่อนซอร์สโค้ด HTML/CSS/JS ที่เผยแพร่ผ่าน GitHub Pages ได้ หากต้องการให้ตัวระบบเป็น Private จริง ควรวางหลังระบบ Access Proxy หรือย้ายไป Hosting/Backend ที่ตรวจสิทธิ์ก่อนส่งหน้าเว็บ

การซ่อนปุ่มสมัครสมาชิกในหน้าเว็บช่วยลดการสมัครโดยไม่ตั้งใจ แต่ไม่ใช่มาตรการความปลอดภัยฝั่งเซิร์ฟเวอร์ ควรจำกัดการสร้างบัญชีจาก Firebase/Identity Platform หรือใช้ขั้นตอนเชิญผู้ใช้โดยผู้ดูแลระบบ

### 2. ตรวจ Firestore Rules ก่อนใช้งานจริง

ต้องยืนยันว่าแต่ละผู้ใช้สามารถอ่าน เขียน และลบได้เฉพาะเอกสารที่ `uid` ตรงกับบัญชีของตน ไม่ควรใช้กฎแบบ `request.auth != null` อย่างเดียว

### 3. เชื่อม Backend หากต้องการ “สร้างสื่อจริง”

เว็บปัจจุบันเป็น Prompt generator เป็นหลัก ถ้าต้องการสร้างภาพ เสียง หรือข้อความจาก AI ภายในเว็บ ควรเรียกผ่าน Backend/Serverless Function และเก็บ API key ฝั่งเซิร์ฟเวอร์เท่านั้น

### 4. ลดหนี้ CSS

`css/style.css` มีแพตช์หลายเวอร์ชันซ้อนกันเกือบ 4,000 บรรทัด ควรแยกเป็นไฟล์ฐาน, component, responsive และ feature เพื่อลดกฎที่ชนกันและทำให้แก้ UI รุ่นต่อไปง่ายขึ้น

### 5. เพิ่มการทดสอบอัตโนมัติ

ควรมี smoke test สำหรับ Login, เปิดเมนูมือถือ, สร้าง Prompt, คัดลอก, บันทึก/ลบโปรเจกต์ และ Album upload รวมถึง Lighthouse/Accessibility test ก่อน deploy ทุกครั้ง

### 6. ทำ PWA ให้ครบหรือถอด Manifest

ตอนนี้มี Web App Manifest แต่ยังไม่มี Service Worker หากต้องการให้ติดตั้งและทำงานออฟไลน์ได้จริง ควรเพิ่ม cache strategy และหน้า offline พร้อมวางแผนอัปเดต cache ตามเวอร์ชัน

## การตั้งค่าที่เพิ่ม

อยู่ใน `js/firebase-config.js`

- `window.TANJAI_ENABLE_LOCAL_FALLBACK = false` — ต้องเป็น `false` บน production
- `window.TANJAI_ALLOW_PUBLIC_REGISTRATION = false` — เปิดเฉพาะเมื่อยอมรับการสมัครจากหน้าเว็บจริง ๆ
